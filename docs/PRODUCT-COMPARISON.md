## 一句话差异化

fe-ai-flow 不是另一个 Cursor / Claude Code、是在它们之上的 **harness 工程化层**。核心价值是让 AI 写代码这件事「自动化 + 流程规范化」、最终保障产出质量。代码具体怎么写 / hunk 怎么改是 Cursor / CC 的事、不是我们要打的战场。

## 飞书文档里有什么

- 用「试课完成消息通知优化」这个真需求作为对比基准
- 三家工具的开发流程图（Mermaid 画板）
- 14 维度对比表（入口形态 / 上下文获取 / 是否分阶段 / HITL 时机 / 数据可审计 / 适合场景 等）
- harness 保障质量的 7 件事（本项目核心能力）
- 现阶段劣势、V1.x / V2 路线推测、待团队对齐的问题
